3. Wichtig: Localhost vs. Online
127.0.0.1 bedeutet immer 'dieses Gerät selbst'. Ein Teilnehmer-Handy, das den QR-Code scannt, kann damit nicht auf den Entwickler-PC zugreifen, außer es läuft auf demselben Gerät. Für echte Nutzung muss der QR-Code eine öffentliche HTTPS-Adresse oder mindestens eine LAN-IP enthalten.
Falsch für echte Teilnehmer:
http://127.0.0.1:3000/join/AB72X

Richtig für Online-Betrieb:
https://fahrduell.de/join/AB72X

Richtig für lokalen WLAN-Test:
http://192.168.178.45:3000/join/AB72X
4. Empfohlene Deployment-Strategie
Für Fahrduell ist Echtzeitfähigkeit entscheidend. Die Plattform sollte daher so aufgebaut werden, dass sie unabhängig von einem einzelnen Anbieter läuft. Empfehlung für Version 1.0: Dockerfähige Next.js/Node-Anwendung mit PostgreSQL und WebSocket-/Socket.IO-Unterstützung.
Codex soll die Anwendung nicht fest auf einen Anbieter verdrahten. Stattdessen soll es ein sauberes Produktionssetup mit Docker, ENV-Variablen und dokumentierten Deploy-Schritten vorbereiten.

6.1 Repository-Audit
☐ Prüfe package.json, Framework-Versionen, Startskripte, Datenbankkonfiguration, Socket-Setup und Authentifizierung.
☐ Dokumentiere, ob SQLite, PostgreSQL, In-Memory-State oder lokale Dateien verwendet werden.

6.2 Environment-Konzept
☐ Erstelle eine .env.example mit allen benötigten Variablen.
☐ Nutze NEXT_PUBLIC_APP_URL für öffentlich sichtbare Links und QR-Codes.
☐ Nutze APP_URL oder SERVER_URL für serverseitige absolute URLs.
☐ Nutze DATABASE_URL für PostgreSQL.
☐ Nutze AUTH_SECRET oder NEXTAUTH_SECRET für Authentifizierung.
☐ Secrets niemals in Git committen.
6.3 QR-Code produktionsfähig machen
☐ QR-Code-Link muss aus NEXT_PUBLIC_APP_URL + /join/[joinCode] erzeugt werden.
☐ Wenn NEXT_PUBLIC_APP_URL fehlt, im Development fallback auf http://localhost:3000 erlauben.
☐ In Produktion darf kein localhost-Link erzeugt werden.
☐ Zeige in der Lobby zusätzlich den Klartext-Link und den Join-Code an.
6.4 Datenbank & Migrationen
☐ Für Produktion PostgreSQL verwenden.
☐ Prisma schema prüfen und Migrationen erzeugen.
☐ Production-Deployment muss npx prisma migrate deploy ausführen.
☐ Optional: Seed-Skript für ersten Admin-User erstellen.
☐ Daten dürfen nach Server-Neustart nicht verloren gehen.
6.5 Realtime / Socket.IO
☐ Socket-Server so konfigurieren, dass er außerhalb von localhost funktioniert.
☐ Allowed Origins aus ENV lesen.
☐ Bei mehreren Server-Instanzen Redis-Adapter vorbereiten oder dokumentieren.
☐ Reconnect-Verhalten testen: Teilnehmer darf bei kurzer Unterbrechung nicht aus dem Spiel fallen.
☐ Room-Struktur pro GameSession sauber trennen.
6.6 Docker & Production Build
☐ Erstelle ein Dockerfile für die App.
☐ Erstelle docker-compose.yml für App + PostgreSQL + optional Redis.
☐ Build-Befehl muss reproduzierbar laufen.
☐ Container muss über PORT konfigurierbar sein.
☐ Healthcheck muss verfügbar sein.
6.7 Security-Basics
☐ HTTPS im Produktivbetrieb erzwingen.
☐ Sichere Cookies in Produktion aktivieren.
☐ Admin-/Dozentenbereiche vor unbefugtem Zugriff schützen.
☐ Join-Codes zufällig und nicht erratbar erzeugen.
☐ Rate-Limits für Login und Join-Endpunkte prüfen.
☐ Keine Secrets oder Stacktraces im Frontend anzeigen.

14. Technische Hinweise / Quellenstand
Stand 26.06.2026: Next.js kann offiziell als Node.js-Server, Docker-Container, statischer Export oder über Adapter bereitgestellt werden. Für Fahrduell sind Node.js/Docker geeigneter als statischer Export, weil Live-Quiz, Auth und Datenbank benötigt werden.
Prisma empfiehlt für Produktionsmigrationen den Befehl npx prisma migrate deploy. Dieser Schritt muss in Deployment/CI berücksichtigt werden.
Socket.IO benötigt bei mehreren Serverinstanzen je nach Konfiguration Sticky Sessions und/oder einen Adapter wie Redis. Für MVP reicht eine einzelne Instanz; für Skalierung muss Redis vorbereitet werden.
Vercel hat WebSocket-Support im Juni 2026 als Public Beta eingeführt. Da die Funktion neu ist, soll Codex die bestehende Socket.IO-Architektur dort gesondert testen. Für die erste stabile Version bleibt Docker/Node auf einer Plattform mit dauerhaftem Node-Prozess die konservativere Empfehlung.
